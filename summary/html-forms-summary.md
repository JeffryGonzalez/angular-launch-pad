# Semantic HTML and Forms Summary

## Key Points about Semantic HTML:
- **Definition**: Semantic HTML refers to the use of HTML markup that conveys meaning about the content enclosed within the tags.
- **Purpose**: Enhances accessibility, search engine optimization, and maintainability of web applications.
- **Common semantic elements**: `header`, `footer`, `article`, `section`, `nav`, and `aside`.

## Key Points about HTML Forms:
- **Definition**: An HTML form is a section of a document that contains interactive controls to submit information to a web server.
- **Common elements**: `<input>`, `<label>`, `<textarea>`, `<select>`, `<button>`, etc.
- **Form attributes**: 
  - `action`: URL to which the form data will be sent.
  - `method`: HTTP method (`GET` or `POST`) used to submit the form.
  - `enctype`: Specifies how the form data should be encoded when submitted.
- **Accessibility considerations**: Ensure all form controls are properly labeled, and use `fieldset` and `legend` elements for grouping related controls.
